因为线程共享一块内存地址，所以当多个线程修改共享资源的时候，就会出现各种各样的问题

输出结果

为什么每次的结果会不一样呢，如果你给的值较小的话，是不会出现这种问题的，只有你的值给的比较大的话，那么就会出现这种问题了，首先我们分析一下，当 Python 解释器遇到 count += 1的时候，其实的分了这么几个步骤的

  • 先将 count += 1 变成 count = count + 1
  • 然后先计算 count + 1
  • 将 count + 1 得到的值重新赋值给 count
    那么应该怎么解决这个问题呢， Python 给我们提供了一个 锁 的东西，那么这个锁应该怎么用，其实也很简单
